Коротка інформація про системи контролю версій та Git

Системи контролю версій (Version Control Systems, VCS) — інструменти, що дозволяють відстежувати зміни в файлах проекту, працювати над однією кодовою базою багатьом користувачам, порівнювати версії та відкотити невдалий код. Git — розподілена VCS, створена для швидкої та ефективної роботи з історією змін, гілками та злиттями. Кожен користувач отримує повну копію репозиторію, що підвищує надійність і дає гнучкість у роботі офлайн.

Основні можливості та переваги використання Git

  • Розподіленість: кожен має локальну копію — швидка робота та можливість працювати офлайн.
  • Швидкість: більшість операцій виконуються локально дуже швидко.
  • Гілки і злиття: просте створення гілок для фіч та паралельної роботи.
  • Історія змін: легкий відкат до попередніх станів і аудит змін.
  • Спільна робота: інтеграція з GitHub/GitLab/Bitbucket робить колаборацію простою.
  • Безпека: контроль цілісності даних через SHA-1 (або інші механізми в сучасних реалізаціях).

10 базових команд Git — з коментарями та прикладами

1) git init
Команда: git init
Коментар: Ініціалізує новий локальний репозиторій у поточній теці. Створює папку .git із метаданими.
Приклад: git init
2) git clone <url>
Команда: git clone https://github.com/user/repo.git
Коментар: Клонує віддалений репозиторій локально — звантажує весь вміст та історію.
3) git add <>файл(и)>
Команда: git add file.txt
Коментар: Додає файли до індексу (staging area) — підготовка до коміту. Для всіх файлів: git add .
4) git commit -m "повідомлення"
Команда: git commit -m "Додано стартовий файл"
Коментар: Фіксує індексні зміни у новий коміт з повідомленням.
5) git push <remote> <branch>
Команда: git push origin main
Коментар: Відправляє локальні коміти на віддалений сервер (remote). Зазвичай origin та назва гілки.
6) git pull
Команда: git pull
Коментар: Забирає останні зміни з віддаленого репозиторію і автоматично зливає їх у поточну гілку (fetch + merge).
7) git status
Команда: git status
Коментар: Показує незафіксовані зміни, файли в індексі та поточну гілку — дуже корисна для орієнтування.
8) git log --oneline
Команда: git log --oneline
Коментар: Показує журнал комітів. Варіант --oneline зручний для стислого перегляду історії.
9) git branch <ім'я_гілки>
Команда: git branch feature-x
Коментар: Створює нову гілку. git branch без параметрів покаже існуючі гілки. Щоб перейти: git checkout feature-x
10) git merge <гілка>
Команда: git merge feature-x
Коментар: Зливає вказану гілку у поточну. Часто використовують після перевірки змін у feature-гілці.
